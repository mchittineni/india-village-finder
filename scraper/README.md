# scraper/ — shared data pipeline

One code path builds the datasets and map apps for **every** state (Andhra Pradesh =
LGD state code `28`, Telangana = `36`, Karnataka = `29`, Tamil Nadu = `33`). Run
everything from this folder.

```bash
python3 -m venv .venv
./.venv/bin/pip install -r requirements-dev.txt   # runtime + pytest
```

## Scripts

| Script                   | What it does                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `pipeline.py`            | Fetches LGD data (districts / sub-districts / villages / pincodes) from the **data.gov.in** open-data API (via `lgd_datagov.py`), filters to each state, **verifies counts against the live LGD portal**, and writes each state's `web/data/*.json` + `data/*.csv`. Writes `web/data/names.json` (authoritative native names — empty when the source lacks the in-script column, as the data.gov.in feed does), copies the UI template, and writes `web/config.js` (incl. the per-state sub-district term — Mandal or Taluk — native language, and the optional `cadastre` block for AP).                                                                                                                                                                                                                  |
| `lgd_datagov.py`         | Pulls LGD village data straight from the official **data.gov.in** OGD API (resource _LGD – Villages with PIN Codes_) with retry/backoff, and writes the four CSVs `pipeline.py` parses. Reads `$DATA_GOV_KEY` (registered free key); falls back to the rate-limited public sample key. On a transient outage it reuses cached CSVs, else signals `DataGovUnavailable` so the caller can skip the run.                                                                                                                                                                                                                                                                                                                                                                                                      |
| `build_boundaries.py`    | Downloads LGD district & sub-district boundary polygons (`ramSeraph/indian_admin_boundaries`), filters per state, drops invalid/duplicate codes, and **simplifies** them (via `mapshaper`) into `web/data/{districts,mandals}.geojson`. Run occasionally — boundaries rarely change.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| `build_parcels_index.py` | **AP only.** Scans the cadastral PMTiles (survey plots) and builds a per-village index — a bounding box (`web/data/parcels_index.json`) and centroid (`web/data/village_points.json`) keyed by LGD village code — so the app can jump precisely to a village's land parcels. Heavy (whole-state tile scan); run in CI via `build-parcels-index.yml`.                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| `enrich_coords.py`       | Best-effort **precise village coordinates**: matches each village by name to a GeoNames place and keeps it only when close to the village's sub-district. Writes `web/data/coords.json`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| `enrich_native_names.py` | Best-effort **neural native names** for the places LGD doesn't publish in-script: runs **AI4Bharat IndicXlit** (a trained English→Indic model, far better than the rule engine). Default writes village names to `web/data/names_translit.json`; `--regions` writes district/sub-district/state names to `web/data/regions_native.json` (reusing same-named villages' native names first, model for the rest — so it degrades gracefully without the model). Run **occasionally** and commit the output — it needs a heavy, **offline-only** dependency (`pip install -r requirements-translit.txt`, pulls in PyTorch). `--eval` scores the model against LGD's authoritative names; `--state ap\|tg\|ka\|tn` limits scope. The map and CSV read the committed JSON, so CI/the browser never need PyTorch. |
| `lgd_client.py`          | Minimal client for LGD's live DWR endpoints. Used by `pipeline.py` for the verification cross-check.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| `changelog.py`           | Compares the working tree to `HEAD` and prints a Markdown summary of data changes — used as the body of the automated refresh PR.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| `release_notes.py`       | Generates the notes + version inputs for the automated GitHub Release.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| `translit_eval.mjs`      | Scores the UI transliteration engine against LGD's authoritative native names (exact-match % + character accuracy). `node scraper/translit_eval.mjs` to report; `--check` enforces a floor (run in CI).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| `translit_cli.mjs`       | Batch transliteration bridge (stdin JSON → native script) so `pipeline.py` can fill the CSV's native-name column using the _same_ engine as the UI (`web_template/i18n.js`).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| `tests/`                 | `pytest` data-validity suite (see the root README).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |

Common flags: `--state ap|tg|ka|tn|both` · `--offline` (reuse cached downloads) ·
`--no-verify` (skip the live LGD check).

## How it fits together

```
lgd_datagov.py ─(data.gov.in LGD API)─> .cache/raw/*.csv ─┐
                                                          ▼
pipeline.py ──> ../<state>/web/data/{regions,villages,names,meta}.json  +  ../<state>/data/*.csv
                 (<state> = andhra_pradesh | telangana | karnataka | tamil_nadu)
build_boundaries.py     ──> ../<state>/web/data/{districts,mandals}.geojson
build_parcels_index.py  ──> ../andhra_pradesh/web/data/{parcels_index,village_points}.json  (AP cadastre)
enrich_coords.py        ──> ../<state>/web/data/coords.json
enrich_native_names.py  ──> ../<state>/web/data/names_translit.json   (offline; IndicXlit; villages)
enrich_native_names.py --regions ──> ../<state>/web/data/regions_native.json  (district/mandal/state)
web_template/ ──(copied by pipeline)──> ../<state>/web/{index.html,app.js,i18n.js,nearby.js,styles.css}
```

`.cache/` holds the downloaded source dumps and is git-ignored (re-downloaded on demand).
Edit the UI **only** in `web_template/`; the per-state copies are regenerated.

**Native village names — layered fallback.** `names.json` holds the **authoritative**
in-script spelling (script-validated); the pipeline still writes it, but the current
LGD source (the data.gov.in feed) doesn't include the local-script column, so it comes
out **empty** today. `enrich_native_names.py` produces a clearly-**approximate** neural
spelling in `names_translit.json`, which is therefore the primary source of native names.
The map (`vname`) and the CSV prefer them in order **authoritative → neural → rule-based
transliteration**, so if a source that publishes official in-script names is wired back
in it automatically wins. The heavy IndicXlit step is optional and run offline, and its
committed output feeds both the map and the CSV; its dependency lives in
`requirements-translit.txt`, intentionally separate from `requirements-dev.txt`.
